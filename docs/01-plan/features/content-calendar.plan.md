# Plan: 컨텐츠 캘린더

## 1. 개요

### 배경
현재 시스템은 YouTube 영상 업로드 → AI 변환 → 각 플랫폼 배포의 "즉시 실행" 플로우만 존재.
콘텐츠의 예약 관리, 진행 상황 추적, 일정 기반 배포가 불가능한 상태.

### 목적
- 콘텐츠 라이프사이클 전체를 캘린더 뷰로 관리
- 예약 배포 기능 제공
- 콘텐츠 상태 추적 (기획 → 제작 → 검수 → 업로드)
- IR deck에서 "콘텐츠 관리 시스템"으로서의 비주얼 임팩트 확보

### 대상 사용자
- 콘텐츠 크리에이터 / 마케팅 담당자
- 멀티 플랫폼 운영하는 1인 미디어 / 소규모 팀

## 2. 기능 요구사항

### FR-01: 월간 캘린더 뷰
- 월간 캘린더에 콘텐츠를 날짜별로 표시
- 각 콘텐츠는 상태별 색상으로 구분
- 월 이동 (이전/다음 월) 네비게이션
- 오늘 날짜 하이라이트

### FR-02: 콘텐츠 상태 관리
- 상태 플로우: `기획중` → `제작중` → `검수필요` → `승인완료` → `업로드 예정` → `업로드 완료`
- 각 상태별 색상:
  - 기획중: 회색
  - 제작중: 파란색
  - 검수필요: 주황색 (경고)
  - 승인완료: 초록색
  - 업로드 예정: 보라색
  - 업로드 완료: 녹색 (진하게)
- 상태 클릭으로 변경 가능 (드롭다운)

### FR-03: 콘텐츠 카드
- 캘린더 셀 안에 콘텐츠 카드 표시
- 카드 정보: 제목, 상태 뱃지, 대상 플랫폼 아이콘들
- 카드 클릭 시 상세 정보 모달

### FR-04: 콘텐츠 추가/수정
- 날짜 클릭으로 새 콘텐츠 추가
- 모달에서 정보 입력: 제목, 설명, 예정일, 대상 플랫폼, 상태
- 기존 콘텐츠 수정 기능

### FR-05: 드래그 앤 드롭 날짜 변경
- 콘텐츠 카드를 드래그하여 다른 날짜로 이동
- 일정 변경의 직관적 UX

### FR-06: 상태 필터
- 상태별 필터링 (특정 상태만 보기)
- 플랫폼별 필터링

## 3. 현재 시스템과의 연결

### 기존 페이지 구조
```
대시보드 → 업로드 → 워크플로우 → 템플릿 → 설정
```

### 변경 후
```
대시보드 → 캘린더 → 업로드 → 워크플로우 → 템플릿 → 설정
```

### 데이터 흐름
1. **캘린더에서 콘텐츠 기획** → 상태: "기획중"
2. **영상 제작 완료** → 상태: "제작중" → "검수필요"
3. **Upload 페이지에서 YouTube URL 입력** → 캘린더 콘텐츠와 연결
4. **Workflow 실행** → 상태: "업로드 예정"
5. **승인 후 배포 완료** → 상태: "업로드 완료"

## 4. 기술 설계 방향

### 프론트엔드
- 새 페이지: `Calendar.tsx`
- 커스텀 캘린더 컴포넌트 (외부 라이브러리 없이 직접 구현 - 가볍게)
- 모달 컴포넌트: `CalendarModal.tsx`
- 드래그 앤 드롭: HTML5 Drag API

### 백엔드
- 기존 contents 테이블에 `scheduled_date`, `content_status` 컬럼 추가
- API 엔드포인트:
  - `GET /api/calendar?year=2026&month=2` - 월별 콘텐츠 조회
  - `PATCH /api/contents/:id/status` - 상태 변경
  - `PATCH /api/contents/:id/schedule` - 날짜 변경

### 상태 정의
```typescript
type ContentStatus =
  | 'planning'        // 기획중
  | 'producing'       // 제작중
  | 'review_needed'   // 검수필요
  | 'approved'        // 승인완료
  | 'scheduled'       // 업로드 예정
  | 'uploaded'        // 업로드 완료
```
